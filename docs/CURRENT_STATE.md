## 当前在做什么

- 项目已从单纯命令行脚本升级为可直接使用的 Web 页面。
- 当前主线目标已经从单一 Amazon 解析扩展到双页面模式：Amazon Summary 与 eBay 财务报告。
- 当前重点转为稳定使用、需求收口和后续小范围迭代。

## 上次已完成

- 支持上传 `zip` / `rar` 压缩包解析。
- 支持直接上传多个 `PDF` 文件解析。
- 页面可展示任务进度、成功数、失败数、跳过数、输出行数和失败明细。
- 页面支持最近任务恢复与任务历史查看。
- `MX` / `BR` 站点已改为“跳过”，不计入失败。
- 零金额未映射字段不会再导致整份文件失败。
- 页面新增“汇率维护”，支持按 `月份 + 站点` 维护美国 / 加拿大汇率。
- 解析时根据 PDF 账期自动匹配对应月份汇率，不再使用代码写死汇率。
- 已新增第二个页面 `/ebay-finance`，支持 eBay 财务报告上传、汇率维护、可选店铺名覆盖、任务历史和刷新恢复。
- eBay 财务报告解析逻辑已迁入当前项目目录，不再依赖外部脚本目录运行。
- 线上服务已部署到 `8.218.58.28:6565`，服务名为 `summary-amazon-pdf`。

## 待办 / 下一步

- 若业务继续扩大，可考虑正式支持 `MX` / `BR`。
- 若需要上线最新 eBay 页面，需要再做一次发版。
- 若要长期稳定使用，可补任务日志下载、用户维度汇率配置、任务持久化增强。

## 注意与坑

- 不要重启服务器或其他项目，只能操作 `summary-amazon-pdf` 这个服务。
- 线上目录为 `/opt/summary-amazon-pdf`。
- 页面汇率默认为空，未维护汇率时不允许开始解析任务。
- 本项目当前需求文档主文件在项目根目录 `requirements.md`。
- eBay 财务报告任务数据保存在 `web_jobs_ebay_finance/`，与 Amazon 的 `web_jobs/` 分离。
